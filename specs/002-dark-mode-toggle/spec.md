# 功能規格：深色模式切換

**功能分支**：`002-dark-mode-toggle`  
**建立日期**：2026-01-29  
**狀態**：草稿  
**輸入**：使用者描述："幫這個待辦事項應用程式加入深色模式切換按鈕。樣式要符合現有設計，並確保切換時所有元件的顏色都能正確對應。"

## 使用者情境與測試 *（必填）*

### 使用者故事 1 - 切換深色模式（優先順序：P1）

使用者想要在使用待辦事項應用程式時，能夠一鍵切換到深色模式，減少眼睛疲勞，特別是在夜間或低光環境下使用時。

**優先順序原因**：這是深色模式功能的核心價值，讓使用者能夠根據環境需求選擇舒適的視覺體驗。

**獨立測試**：可透過點擊切換按鈕完整測試，驗證所有介面元素是否正確切換顏色，交付「在低光環境下舒適使用」的價值。

**驗收情境**：

1. **假設** 應用程式處於淺色模式，**當** 使用者點擊深色模式切換按鈕，**則** 介面立即切換為深色主題，所有元件（背景、文字、按鈕、輸入框、待辦項目卡片）都呈現對應的深色配色
2. **假設** 應用程式處於深色模式，**當** 使用者點擊切換按鈕，**則** 介面切換回淺色主題，恢復原本的配色
3. **假設** 使用者使用螢幕閱讀器，**當** 聚焦到切換按鈕時，**則** 能聽到當前模式狀態與按鈕用途的描述

---

### 使用者故事 2 - 記住使用者偏好（優先順序：P2）

使用者希望應用程式能記住他們選擇的顏色模式，下次開啟時自動套用，不需要每次都重新設定。

**優先順序原因**：提升使用者體驗的延續性，避免重複操作，但核心切換功能更為重要。

**獨立測試**：可透過設定偏好後重新載入頁面來測試，驗證設定是否正確保存和恢復。

**驗收情境**：

1. **假設** 使用者將模式切換為深色，**當** 關閉並重新開啟應用程式，**則** 應用程式自動以深色模式呈現
2. **假設** 使用者將模式切換為淺色，**當** 重新載入頁面，**則** 應用程式保持淺色模式
3. **假設** 使用者首次使用應用程式且系統為深色模式，**當** 開啟應用程式，**則** 自動偵測並套用深色模式
4. **假設** 使用者首次使用應用程式且系統為淺色模式，**當** 開啟應用程式，**則** 自動偵測並套用淺色模式

---

### 使用者故事 3 - 切換按鈕的視覺識別（優先順序：P1）

使用者需要能夠輕易找到並識別深色模式切換按鈕，按鈕的視覺設計應符合現有介面風格。

**優先順序原因**：與切換功能同等重要，使用者必須能找到按鈕才能使用功能。

**獨立測試**：可透過視覺檢查與使用者互動測試，驗證按鈕可見性、位置合理性與圖示意義清晰度。

**驗收情境**：

1. **假設** 使用者開啟應用程式，**當** 查看標題區域，**則** 能看到一個直覺的深色/淺色模式切換按鈕
2. **假設** 應用程式處於淺色模式，**當** 查看切換按鈕，**則** 按鈕顯示代表「切換到深色」的圖示（如月亮符號）
3. **假設** 應用程式處於深色模式，**當** 查看切換按鈕，**則** 按鈕顯示代表「切換到淺色」的圖示（如太陽符號）

### 邊界情境

- 當使用者的瀏覽器不支援 localStorage 時會發生什麼？
  - 系統應正常運作，切換功能可用，但偏好設定不會被保存
- 當切換按鈕在行動裝置上顯示時，觸控目標是否足夠大？
  - 按鈕應符合無障礙標準，觸控目標至少 44x44 像素
- 當使用者快速連續點擊切換按鈕時會發生什麼？
  - 系統應正確處理每次點擊，不會出現視覺閃爍或狀態錯亂

## 需求 *（必填）*

### 功能性需求

- **FR-001**：系統必須在標題區域提供一個深色模式切換按鈕
- **FR-002**：系統必須支援淺色與深色兩種主題模式的即時切換
- **FR-003**：切換按鈕必須顯示代表當前模式的圖示（淺色模式顯示月亮，深色模式顯示太陽）
- **FR-004**：系統必須將使用者的顏色模式偏好保存在本地儲存中
- **FR-005**：系統必須在頁面載入時讀取並套用已保存的顏色模式偏好
- **FR-006**：深色模式必須覆蓋所有視覺元件，包括：
  - 頁面背景
  - 標題文字
  - 輸入框（背景、文字、邊框、佔位符）
  - 按鈕（主要、次要、危險、圖示）
  - 待辦項目卡片（背景、文字、邊框）
  - 群組標題
  - 空狀態提示
  - 字元計數器
  - 錯誤訊息
- **FR-007**：切換按鈕必須具備適當的無障礙標籤，說明其用途與當前狀態
- **FR-008**：切換過程必須有平滑的視覺過渡效果

### 關鍵實體

- **顏色模式偏好**：代表使用者選擇的主題模式（淺色/深色），保存於本地儲存中

## 成功標準 *（必填）*

### 可衡量的成果

- **SC-001**：使用者點擊切換按鈕後，所有介面元素在 0.3 秒內完成顏色轉換
- **SC-002**：100% 的可見介面元件在深色模式下具有適當的對比度（文字與背景對比度符合 WCAG AA 標準，至少 4.5:1）
- **SC-003**：使用者偏好設定在重新載入頁面後 100% 正確恢復
- **SC-004**：切換按鈕的觸控目標在所有裝置上至少達到 44x44 像素
- **SC-005**：螢幕閱讀器使用者能清楚識別切換按鈕的用途與當前狀態

## 假設

- 現有應用程式使用 CSS 變數定義顏色，可透過修改 CSS 變數值實現主題切換
- 目標瀏覽器支援 CSS 變數和 localStorage
- 使用者對「月亮代表深色」和「太陽代表淺色」的圖示隱喻有普遍認知
- 首次使用時自動偵測系統偏好（`prefers-color-scheme`），若系統為深色模式則預設深色，否則預設淺色
